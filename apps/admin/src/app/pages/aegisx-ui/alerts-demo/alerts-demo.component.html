Best Practices

üí° Usage Guidelines

‚úÖ Do
  • ‚Ä¢ Use appropriate variant colors (success for confirmations, error for failures)
  • ‚Ä¢ Keep alert messages concise and actionable
  • ‚Ä¢ Use titles for complex or important alerts
  • ‚Ä¢ Make alerts closeable for non-critical messages
  • ‚Ä¢ Use auto-hide for temporary notifications (3-10 seconds)
  • ‚Ä¢ Stack multiple alerts vertically for better readability
‚ùå Don't
  • ‚Ä¢ Don't use alerts for every minor action (causes alert fatigue)
  • ‚Ä¢ Don't write long paragraphs in alerts (keep them brief)
  • ‚Ä¢ Don't mix alert types inappropriately (e.g., success for errors)
  • ‚Ä¢ Don't make critical error alerts auto-hide (users need time to read)
  • ‚Ä¢ Don't show multiple alerts for the same event
  • ‚Ä¢ Don't use alerts as primary navigation or content containers
üìä Recommended Durations
  • ‚Ä¢ Success messages: 3-5 seconds
  • ‚Ä¢ Info notifications: 5-7 seconds
  • ‚Ä¢ Warnings: 7-10 seconds
  • ‚Ä¢ Errors: Don't auto-hide (or 10+ seconds)
