Secondary Heading with Border

H2 headings include a bottom border to clearly separate major sections. They use 30px font size with semibold weight (600) and snug line height (1.3) for compact yet readable section headers.

Tertiary Heading Level 3

H3 headings are 24px with semibold weight, perfect for subsections within major sections. They maintain visual hierarchy while keeping content scannable.

Heading Level 4 - Subsection

H4 headings use 20px size and are ideal for smaller subsections or topic divisions within subsections.

Heading Level 5

H5 uses the base text size (15px) with semibold weight for subtle section headers.

Heading Level 6

H6 is the smallest heading at 14px, useful for minor subsections or labels.

view_column Prose Container Variants

Default Prose (65ch max-width)

The default .prose container has a maximum width of 65 characters (~700-750px). This is the optimal line length for reading, based on typography research. Lines that are too long make it difficult for readers to track from the end of one line to the beginning of the next.

Wide Prose (75ch max-width)

The .prose-wide variant increases the maximum width to 75 characters, providing slightly more horizontal space while still maintaining readability. Use this for content that benefits from wider columns, such as code-heavy documentation or comparison tables.

Full Width Prose (no max-width)

The .prose-full variant removes the maximum width constraint entirely, allowing content to expand to fill its container. This is useful for special layouts, dashboards, or when you want complete control over the width through other means.
